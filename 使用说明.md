# 课堂评分小程序使用说明

## 结论

如果学生端和教师端不在同一个无线网，不能只用教师电脑的局域网地址。需要让两个页面都访问同一个公网地址。

推荐做法：

1. 把本程序部署到公网服务器。
2. 教师打开 `https://你的公网地址/teacher`。
3. 教师页会显示“发送给学生的评分网址”，复制后发给全班。
4. 学生打开该网址提交匿名整数评分。
5. 教师页会实时更新各小组平均分，右下角显示当前选中小组的最终得分。

## 本地同一网络使用

在本文件夹运行：

```powershell
npm start
```

教师页：

```text
http://localhost:3000/teacher
```

学生页：

```text
http://教师电脑IP:3000/
```

这种方式要求学生设备和教师电脑在同一个局域网中。

## 不同网络使用

需要部署到公网。可用 Render、Railway、腾讯云、阿里云、学校服务器等 Node.js 环境。

部署后访问：

```text
https://你的公网地址/
https://你的公网地址/teacher
```

## 教师 PIN

公网部署时建议设置环境变量：

```text
TEACHER_PIN=你自己设定的密码
```

设置后，教师页清空全部评分时需要输入这个 PIN。

## 数据说明

- 学生匿名提交，不填写姓名。
- 每台设备对同一小组重复提交时，会更新该设备自己的评分。
- 分数只能是 `0-10` 的整数。
- 数据保存在服务端的 `scores.json`。
- 如果部署平台重启后不保留文件，需要在正式上课前确认数据持久化设置。
